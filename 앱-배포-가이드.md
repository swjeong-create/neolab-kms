# OCI 인스턴스 앱 배포 가이드

이 문서는 OCI(Oracle Cloud Infrastructure)에 생성된 인스턴스에 웹앱을 배포하는 방법을 안내합니다.

## 서버 접속 정보

인프라 담당자로부터 아래 정보를 전달받아야 합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| IP 주소 | 서버의 Public IP | `130.162.139.142` |
| SSH 포트 | 기본 22가 아닌 **22022** 사용 | `22022` |
| SSH 키 파일 | 전달받은 private key 파일 | `deploy-key-infra-kms` |
| 사용자 | Ubuntu 리눅스 기본 계정 | `ubuntu` |
| 앱 디렉토리 | 코드가 위치할 경로 | `/opt/앱이름` |

## 1단계: SSH 키 파일 준비

전달받은 SSH 키 파일을 `~/.ssh/` 디렉토리에 저장하고 권한을 설정합니다.

```bash
# Windows (PowerShell)
copy deploy-key-infra-kms $env:USERPROFILE\.ssh\deploy-key-infra-kms

# Mac/Linux
cp deploy-key-infra-kms ~/.ssh/deploy-key-infra-kms
chmod 600 ~/.ssh/deploy-key-infra-kms
```

## 2단계: SSH 접속 확인

```bash
ssh -i ~/.ssh/deploy-key-infra-kms -p 22022 ubuntu@[IP주소]
```

접속이 되면 `ubuntu@인스턴스명:~$` 프롬프트가 나타납니다.

## 3단계: 서버 환경 확인

인프라 담당자가 아래 항목을 미리 설치해둔 상태입니다.

```bash
node -v     # Node.js 버전 확인
pm2 -v      # PM2 (프로세스 매니저) 버전 확인
```

## 4단계: 코드 배포

### 방법 A: Git Clone (권장)

인프라 담당자가 이미 clone해둔 경우:

```bash
cd /opt/앱이름
git pull origin main
npm install
```

처음부터 하는 경우:

```bash
cd /opt
sudo git clone https://github.com/계정/저장소.git 앱이름
sudo chown -R ubuntu:ubuntu /opt/앱이름
cd /opt/앱이름
npm install
```

### 방법 B: 파일 직접 전송 (scp)

로컬에서 서버로 파일을 보내는 방법:

```bash
# 단일 파일
scp -i ~/.ssh/deploy-key-infra-kms -P 22022 ./파일명 ubuntu@[IP주소]:/opt/앱이름/

# 폴더 전체
scp -i ~/.ssh/deploy-key-infra-kms -P 22022 -r ./폴더명 ubuntu@[IP주소]:/opt/앱이름/
```

## 5단계: 환경 변수 설정

`.env` 파일을 서버에 생성합니다.

```bash
cd /opt/앱이름
nano .env
```

> `nano`는 터미널 텍스트 편집기입니다.
> - 편집 후 `Ctrl + O` → `Enter`로 저장
> - `Ctrl + X`로 종료

또는 로컬에서 직접 전송:

```bash
scp -i ~/.ssh/deploy-key-infra-kms -P 22022 ./.env ubuntu@[IP주소]:/opt/앱이름/.env
```

인증 키 파일(예: `service-account.json`)도 같은 방법으로 전송합니다.

## 6단계: 앱 실행

```bash
cd /opt/앱이름

# PM2로 실행 (백그라운드 + 자동 재시작)
pm2 start server.js --name 앱이름

# 실행 확인
pm2 status

# 로그 확인
pm2 logs 앱이름
```

## 7단계: 브라우저에서 확인

```
http://[IP주소]:3000
```

> HTTPS가 필요한 경우 인프라 담당자에게 nginx 또는 Cloudflare Tunnel 설정을 요청하세요.

## 자주 쓰는 명령어

| 명령어 | 설명 |
|--------|------|
| `pm2 status` | 실행 중인 앱 목록 |
| `pm2 logs 앱이름` | 실시간 로그 보기 |
| `pm2 restart 앱이름` | 앱 재시작 |
| `pm2 stop 앱이름` | 앱 중지 |
| `cd /opt/앱이름 && git pull && npm install && pm2 restart 앱이름` | 코드 업데이트 후 재시작 |

## 문제가 생겼을 때

| 증상 | 확인 방법 |
|------|----------|
| SSH 접속 안 됨 | 키 파일 권한 확인, 포트 22022 맞는지 확인 |
| 앱이 안 뜸 | `pm2 logs 앱이름`으로 에러 확인 |
| 포트 접근 안 됨 | 인프라 담당자에게 방화벽(Security List) 확인 요청 |
| 메모리 부족 | `free -h`로 확인, 인프라 담당자에게 스펙 변경 요청 |
